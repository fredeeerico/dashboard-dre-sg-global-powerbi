# Modelo de README para Projetos de Portfólio

Use este modelo para manter todos os projetos do portfólio com a mesma estrutura.

```markdown
# Título do Projeto
### Subtítulo técnico

Case Study — Área do projeto com ferramentas principais

Tecnologia 1 • Tecnologia 2 • Tecnologia 3 • Competência 1 • Competência 2

Descrição curta do projeto em 2 ou 3 linhas, explicando o problema, a solução e o valor gerado.

![Dashboard Preview](assets/dashboard-preview.png)

---

# Objetivo do Projeto

Explique o objetivo de negócio e o que o projeto resolve.

---

# Meu Papel no Projeto

Informe se foi líder, participante, execução individual ou responsável por uma etapa específica.

---

# Teste Rápido

Mostre como abrir, executar ou validar o projeto.

---

# Saiba Mais

Inclua o fluxo do projeto.

---

# Principais Funcionalidades

Liste as funcionalidades principais.

---

# Resultados Obtidos

Inclua KPIs e resultados mensuráveis.

---

# Estrutura do Repositório

Mostre a árvore de pastas.

---

# Explore o Projeto

Inclua links para documentos internos.

---

# Tecnologias Utilizadas

Liste as ferramentas e tecnologias.

---

# Sobre os Dados

Explique a origem, limitações e cuidados de privacidade.

---

# Competências Demonstradas

Liste competências técnicas e analíticas.
```

## Checklist antes de publicar

- [ ] README principal claro e direto
- [ ] Imagem de preview no topo
- [ ] Dashboard em pasta separada
- [ ] Dados brutos e tratados organizados
- [ ] Documentação em Markdown
- [ ] Script de reprocessamento quando aplicável
- [ ] `.gitignore` criado
- [ ] Arquivos sensíveis removidos
- [ ] Repositório testado no GitHub
